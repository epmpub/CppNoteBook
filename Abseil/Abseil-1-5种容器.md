5 种容器

1. 95% 场景用：flat_hash_map /flat_hash_set（最快、最小、连续内存）
2. 需要稳定迭代器 / 指针不失效：node_hash_map /node_hash_set
3. 需要有序 / 范围查询：btree_map /btree_set

# 2. 逐类详细差异（最关键）

## ① flat_hash_map /flat_hash_set（默认首选！）

### 本质

**连续数组 + 开放寻址**（就是你之前问的 Swiss Table）

### 优点

- **查找 / 遍历 / 访问速度最快**
- **内存占用最小**
- CPU 缓存命中率极高
- 无内存碎片

### 缺点

- **扩容时，所有迭代器、指针全部失效**
- 不适合超大对象（> 1KB）

### 适用场景

- 高频查找
- 读多写少
- 遍历频繁
- 小对象（int /string/ 指针）
- **95% 的业务都应该首选它**

------

## ② node_hash_map /node_hash_set（稳定版哈希）

### 本质

**每个元素独立分配内存（节点式）+ 哈希表索引**

类似 `std::unordered_map`，但更快。

### 优点

- **迭代器、元素指针、引用 非常稳定**

  

  插入、删除 

  不会失效

   其他元素的指针

- 适合存放**大对象**

- 性能依然远超 std::unordered_map

### 缺点

- 比 flat_hash_* 慢 20%~50%
- 内存占用更高
- 缓存不友好

### 适用场景

- 元素**不能移动 / 拷贝**
- 需要长期持有元素**指针 / 引用**
- 大对象存储

------

## ③ btree_map /btree_set（有序结构）

### 本质

**B 树（平衡多路搜索树）**

**有序！有序！有序！**

### 优点

- **key 自动有序排列**
- 支持**范围查询**（lower_bound / upper_bound）
- 内存占用比 std::map 小很多
- 极高并发遍历性能

### 缺点

- 查找速度 **远慢于哈希表**
- 插入速度慢于哈希

### 适用场景

- **必须有序遍历**
- 需要范围查询（> < between）
- 按 key 排序输出

------

**1. flat_hash_map（最快）**

2. node_hash_map
3. btree_map（最慢，但有序）

------

# 4. 内存占用排名（从小到大）

**1. flat_hash_map（最小）**

2. node_hash_map
3. btree_map

------

# 5. 迭代器 / 指针稳定性（最重要差异）

## flat_hash_map

❌ **插入 / 扩容 → 全部迭代器失效**

❌ 元素地址会变

## node_hash_map

✅ **插入 / 删除 → 其他迭代器都有效**

✅ 元素地址永远不变

## btree_map

✅ **稳定迭代器**

------

# 6. 最实用的选择指南（工程直接用）

## 你不知道选啥？

👉 **直接用 flat_hash_map /flat_hash_set**

## 你需要存大对象 / 持有元素指针？

👉 **node_hash_map / node_hash_set**

## 你需要 key 有序 / 范围查找？

👉 **btree_map / btree_set**

------

# 7. 极简总结表（一眼看懂）

|       容器        | 存储结构 |  速度  |  内存  | 迭代器稳定性 | 有序 |      首选场景      |
| :---------------: | :------: | :----: | :----: | :----------: | :--: | :----------------: |
| **flat_hash_map** | 连续数组 | 🥇 最快 | 🥇 最小 |   ❌ 不稳定   |  ❌   |    95% 通用场景    |
| **node_hash_map** |   节点   |  🥈 快  | 🥈 中等 |    ✅ 稳定    |  ❌   | 大对象、指针不失效 |
|   **btree_map**   |   B 树   |  🥉 慢  | 🥉 较大 |    ✅ 稳定    |  ✅   |   有序、范围查询   |

------

### 最终总结

- **flat 系列 = 最快最小（默认）**
- **node 系列 = 稳定不失效（特殊需求）**
- **btree 系列 = 有序范围查询（必须有序才用）**
