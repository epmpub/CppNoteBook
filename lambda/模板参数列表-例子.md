





下面给出一个完整可编译运行的例子，演示截图中的三个关键点：

1. 泛型 Lambda 的模板参数列表 `[]<typename T>`
2. 调用类型的静态成员函数 `T::static_function()`
3. 访问嵌套类型 `typename T::iterator`

```cpp
#include <iostream>
#include <vector>
#include <string>

// 一个满足要求的类型
struct MyContainer
{
    using iterator = std::vector<int>::iterator;

    static void static_function()
    {
        std::cout << "MyContainer::static_function()\n";
    }

    std::vector<int> data{1, 2, 3, 4};
};

int main()
{
    auto f = []<typename T>(T const& x)
    {
        // 1. 拷贝对象
        T copy = x;

        std::cout << "copy created\n";

        // 2. 调用静态成员函数
        T::static_function();

        // 3. 使用嵌套类型
        using Iterator = typename T::iterator;

        Iterator it = copy.data.begin();

        std::cout << "first element = "
                  << *it
                  << '\n';

        std::cout << "iterator type acquired successfully\n";
    };

    MyContainer c;

    f(c);
}
```

输出：

```text
copy created
MyContainer::static_function()
first element = 1
iterator type acquired successfully
```

------

如果想更明显地看到 Lambda 本质上是一个函数模板，可以显式指定模板参数：

```cpp
f.operator()<MyContainer>(c);
```

与

```cpp
f(c);
```

完全等价。

编译器实际上生成的东西接近于：

```cpp
class Closure
{
public:
    template<typename T>
    void operator()(T const& x) const
    {
        T copy = x;
        T::static_function();

        using Iterator = typename T::iterator;

        Iterator it = copy.data.begin();

        std::cout << *it << '\n';
    }
};
```

------

再进一步，可以利用 C++20 Concept 对 `T` 加约束：

```cpp
#include <concepts>

template<typename T>
concept ContainerLike =
requires
{
    typename T::iterator;
    T::static_function();
};

auto f = []<ContainerLike T>(T const& x)
{
    T::static_function();

    using Iterator = typename T::iterator;

    std::cout << "ContainerLike type\n";
};
```

这样只有同时拥有：

```cpp
typename T::iterator
T::static_function()
```

的类型才能调用这个 Lambda。

这也是 C++20 泛型 Lambda 最强大的地方之一：**Lambda 已经具备了与函数模板几乎相同的表达能力。**
