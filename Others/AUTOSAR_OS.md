

### 一、核心基础：ISO 17356（OSEK/VDX）

AUTOSAR OS 以**OSEK OS 2.2.3**为基础，该标准已被**ISO 17356**系列正式标准化。

- **ISO 17356-1**：通用结构、术语与定义
- **ISO 17356-2**：OSEK OS（操作系统规范）
- **ISO 17356-3**：OSEK COM（通信规范）
- **ISO 17356-4**：OSEKtime（时间触发扩展）
- **ISO 17356-5**：OSEK NM（网络管理规范）

### 二、功能安全：ISO 26262

汽车电子功能安全强制标准，AUTOSAR OS 需支持**ASIL A~D**等级。

- 核心要求：**内存保护、时间保护、任务监控、错误检测与处理、自由干涉（Freedom from ）**。
- 实现机制：MPU/MMU 隔离、优先级天花板、死锁检测、堆栈溢出检查、看门狗管理。

### 三、网络安全：ISO 21434

汽车网络安全工程标准，AUTOSAR OS 需提供安全基础。

- 核心要求：**安全启动、安全更新、访问控制、加密服务、安全日志、漏洞管理**。
- 实现机制：与**Crypto、Secure Boot、HSM**等模块协同，保障 OS 自身与通信安全。

### 四、其他相关 ISO 规范

- **ISO 11898（CAN）**：车载 CAN 通信基础，OS 需适配 CAN 调度与中断处理AUTOSAR。
- **ISO 15118（V2G）**：车网互动通信，OS 需支持相关协议栈调度。
- **ISO/IEC 9899（C 语言）**：AUTOSAR OS 基于 C 语言开发，需遵循 C99/C11 标准AUTOSAR。

### 五、规范关系总览

| ISO 规范         | 核心作用               | AUTOSAR OS 对应要求                     |
| ---------------- | ---------------------- | --------------------------------------- |
| **ISO 17356**    | 定义 OS 基础接口与行为 | 任务调度、资源管理、中断处理、OSEK 兼容 |
| **ISO 26262**    | 保障功能安全           | 内存 / 时间保护、错误检测、ASIL 支持    |
| **ISO 21434**    | 保障网络安全           | 安全启动、访问控制、加密服务            |
| **ISO 11898**    | 定义 CAN 通信          | 适配 CAN 中断、调度与通信栈             |
| **ISO/IEC 9899** | 定义 C 语言标准        | 代码合规、编译器适配                    |
