### `std::make_shared` 的数组支持

指的是从 C++20 开始，它不再只用于创建单个对象，也可以直接创建 `shared_ptr` 管理的动态数组 。这让“创建对象 + 分配控制块”这件事对数组也变得统一了 。

## C++20 之前

在 C++20 之前，`std::make_shared<T>(...)` 主要只支持非数组类型；如果你要管理数组，通常得自己写 `new[]`，再配合自定义删除器，或者改用其他方案 。
比如下面这种写法在早期更常见：

```
cpp
std::shared_ptr<int> p(new int[10], [](int* x){ delete[] x; });
```

这种方式能用，但可读性和安全性都不如直接工厂函数方便 。

## C++20 新增用法

C++20 增加了对数组的重载，常见形式包括：

```c
auto p1 = std::make_shared<int[]>(10);
auto p2 = std::make_shared<int[5]>();
```

其中，`int[]` 表示运行时长度数组，`int[5]` 表示固定长度数组 。
如果是 `int[]`，你需要提供元素个数；如果是 `int[N]`，长度已经写在类型里了 

## 行为特点

数组元素会像普通动态数组一样被 `shared_ptr` 管理，引用计数归零时自动释放，不需要你手动 `delete[]` 
默认初始化时，像 `int` 这样的内置类型元素通常会被值初始化为 0；如果是类类型，则会调用相应构造函数 。
访问元素时依然可以用下标，但要注意 `shared_ptr` 本身不做越界检查 。

## 适合场景

这个特性特别适合需要共享所有权的动态缓冲区，比如图像像素数组、临时缓存、批处理数据等 。
它的优势是代码更短、异常安全更好，也减少了手写删除器出错的概率 

## 注意事项

- 它解决的是“`shared_ptr` 管数组”的问题，不是 `std::vector` 的替代品；如果你只是想要可变长度容器，`vector` 往往更合适 
- 如果你不需要共享所有权，只是单独拥有数组，`std::unique_ptr<T[]>` 也可能更轻量 。
- `make_shared_for_overwrite` 也是 C++20 的相关增强，适合想跳过默认初始化、追求性能的场景 。
