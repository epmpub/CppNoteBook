## 一、P2077：关联容器的异质删除重载（Heterogeneous erasure）

### 背景

C++ 早就支持"异质查找"（heterogeneous lookup）——如果关联容器（`map`、`set` 及其 `unordered_` 版本）使用**透明比较器**（transparent comparator，如 `std::less<>`），就可以用一个"可比较但类型不同"的键去查找，而不必先构造一个临时的 `key_type` 对象：

```cpp
std::map<std::string, int, std::less<>> m = {{"hello", 1}, {"world", 2}};

// C++14 起，用 const char* 直接查找，不需要构造临时 std::string
auto it = m.find("hello");   // 异质查找，无临时对象
```

但问题是：`erase()` 一直没有对应的异质版本！C++23 之前，如果你想删除，只能：

```cpp
m.erase("hello");  
// 在 C++23 之前，如果只有 erase(const key_type&) 和 erase(iterator) 重载
// "hello" 会被隐式转换成一个临时 std::string 对象，再传给 erase(const key_type&)
// 这个临时对象的构造/析构完全是不必要的开销
```

### C++23 的改动

新增了异质 `erase` 重载（以 `std::map` 为例）：

```cpp
namespace std {
    template<class Key, class T, class Compare, class Allocator>
    class map {
    public:
        // 已有
        size_type erase(const key_type& x);
        iterator erase(iterator position);

        // C++23 新增：异质删除
        template<class K>
        size_type erase(K&& x);   // 仅当 Compare 是透明比较器，且 K 不会歧义匹配已有重载时才参与重载决议
    };
}
```

现在：

```cpp
std::map<std::string, int, std::less<>> m = {{"hello", 1}, {"world", 2}};

m.erase("hello");  
// C++23 起：直接用 const char* 完成比较和删除，不再构造临时 std::string
```

### 性能提升

根据 P2077 论文的测试数据，对 `unordered_map`，通过消除临时对象构造，删除操作性能提升约 **20%**；对 `unordered_multimap`，由于原来需要对 `equal_range` 遍历两次（一次定位范围，一次实际删除），改进后额外还能再提升约 **10%**。

### 适用容器

`map`、`multimap`、`set`、`multiset`、`unordered_map`、`unordered_multimap`、`unordered_set`、`unordered_multiset` 均获得了这个重载（前提是使用透明比较器/哈希）。

### 参数为什么用 `K&&`（转发引用）而非 `const K&`

论文中提到，如果直接照搬异质查找的接口用 `const K&`，会丢失值类别（value category）信息，导致某些重载决议场景出问题。用转发引用 `K&&` 可以保留传入参数的值类别，并且能保证该重载**只在没有更好的非模板重载匹配时才参与决议**，从而不会破坏已有代码的行为
