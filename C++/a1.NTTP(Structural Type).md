在 C++20 中，Structural Type（结构化类型）

是一个专门为 NTTP（Non-Type Template Parameter，非类型模板参数）扩展引入的概念。

它的作用是：

“让编译器能够在编译期完全比较两个对象的值是否相同。”

因为模板参数必须能够：

- 编译期确定
- 编译期比较
- 编译期哈希/唯一化
- 参与模板实例化

所以 C++20 对“哪些类可以作为 NTTP”做了严格限制，这类满足条件的类型就叫 structural type。

最典型例子：

```cpp
template<auto V>
struct X {};
```

以前只能：

```cpp
X<123>
X<'A'>
X<&func>
```

C++20 后允许：

```cpp
struct Point
{
    int x;
    int y;
};

template<Point P>
struct Test {};
```

因为 `Point` 是 structural type。

例如：

```cpp
#include <iostream>

struct Point
{
    int x;
    int y;

    constexpr bool operator==(const Point&) const = default;
};

template<Point P>
struct Test
{
    static void print()
    {
        std::cout << P.x << ", " << P.y << "\n";
    }
};

int main()
{
    Test<Point{1,2}>::print();
}
```

这里：

```cpp
Point{1,2}
```

成为了模板参数。

关键问题是：

编译器如何判断：

```cpp
Point{1,2}
```

和：

```cpp
Point{1,2}
```

是不是同一个模板实例？

Structural type 的规则就是为了解决这个问题。

C++20 大致要求：

一个类要成为 structural type，必须：

1. 是 literal type
2. 所有基类和成员都是 public
3. 所有成员自身也必须是 structural type
4. 不能有 mutable 成员
5. 不能有虚函数
6. 不能有虚基类
7. 通常需要默认比较（或可比较）

例如：

合法：

```cpp
struct Point
{
    int x;
    int y;
};
```

因为：

- int 是 structural type
- 成员 public
- 无虚函数
- 无 private

非法：

```cpp
struct Bad
{
private:
    int x;
};
```

因为 private 成员会影响对象语义，编译器不能简单做“结构化比较”。

再例如：

```cpp
struct Bad
{
    std::string s;
};
```

非法，因为：

```cpp
std::string
```

不是 structural type。

因为它内部：

- 动态内存
- allocator
- 指针
- 运行期状态

无法纯编译期结构化比较。

而：

```cpp
std::string_view
```

虽然比 string 简单，但仍然不是 structural type。

原因主要是：

```cpp
struct string_view
{
    const char* ptr;
    size_t size;
};
```

这里存在：

```cpp
const char*
```

而指针比较涉及：

- 地址身份
- 对象生命周期
- 不同翻译单元
- ABI

标准库没有保证它可作为 structural type。

因此：

```cpp
template<std::string_view S>
```

非法。

而 fixed_string：

```cpp
template<size_t N>
struct fixed_string
{
    char data[N];
};
```

是 structural type，因为：

- char[N] 是 structural
- 无 private
- 无动态内存
- 编译器可逐字节比较

本质上：

Structural Type 可以理解为：

“编译器能够完全展开并逐成员比较的纯值类型（pure value type）。”

它有点类似：

- Rust const generics 的可比较 const value
- Haskell type-level literal
- 编译期 immutable value object

这是 C++ 从：

“模板只能接受整数/指针”

向：

“模板可以接受真正的值对象”

演进的重要一步。
