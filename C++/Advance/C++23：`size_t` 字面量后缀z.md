# C++23：`size_t` 字面量后缀z（P0330R8）

## 背景问题

在 C++23 之前，标准库并没有为 `std::size_t`（无符号）或"有符号版本的 `size_t`"提供专门的整数字面量后缀。写代码时如果想让一个字面量的类型明确是 `size_t`，只能依赖平台相关的类型别名去猜后缀：

```cpp
auto n = 42u;    // unsigned int，但 size_t 在 64 位平台通常是 unsigned long / unsigned long long，类型不一定匹配
auto n = 42ul;   // unsigned long，也不一定和 size_t 一致（取决于平台/ABI）
auto n = 42ull;  // unsigned long long，同样不保证等于 size_t
```

这就导致一个常见的痛点：**想要一个"确定是 `size_t` 类型"的字面量，却没有可移植的写法**，只能用 `static_cast<std::size_t>(42)` 这种啰嗦写法，或者依赖某个具体平台的类型恰好匹配。

同时，容器的 `.size()` 返回 `size_t`（无符号），而循环变量、下标计算等场景经常需要**有符号**类型来避免无符号运算下溢（比如 `for (size_t i = v.size() - 1; i >= 0; --i)` 在 `v` 为空时是经典 bug，因为 `size_t` 是无符号，永远 `>= 0`），标准库却一直没有提供"有符号 size_t"的标准字面量或类型别名。

## C++23 的解决方案：`z` / `Z` 后缀

C++23 新增了字面量后缀 `z`（小写）或 `Z`（大写），规则如下：

- 对于**无符号**整数字面量：`uz`、`uZ`、`Uz`、`UZ`（`u`/`U` 与 `z`/`Z`可以任意顺序组合）→ 类型为 `std::size_t`
- 对于**有符号**整数字面量：单独的 `z` 或 `Z` → 类型为**与 `std::size_t` 对应的有符号整数类型**（标准称为 "signed size_t"，实践中通常是 `std::make_signed_t<std::size_t>`，在多数平台上等价于 `std::ptrdiff_t` / `ssize_t`，但标准并不强制两者必须是同一类型，只保证宽度和 `size_t` 相同）

```cpp
auto a = 42uz;   // 类型：std::size_t
auto b = 42Uz;   // 同上
auto c = 42z;    // 类型：有符号版本的 size_t（例如 ptrdiff_t 或等宽有符号类型）
auto d = 42Z;    // 同上
```

## 典型应用场景

### 1. 明确表达"这是一个 size_t 值"

```cpp
std::vector<int> v = {1, 2, 3, 4, 5};

// C++23 之前：类型不一定精确匹配 size_t
for (auto i = 0u; i < v.size(); ++i) { ... }

// C++23：明确、可移植地表达 size_t 类型
for (auto i = 0uz; i < v.size(); ++i) { ... }
```

### 2. 避免无符号下溢陷阱（这是最实用的场景）

经典 bug：

```cpp
std::vector<int> v; // 空
for (std::size_t i = v.size() - 1; i >= 0; --i) { // v.size()-1 下溢为一个巨大的数，死循环/越界
    std::cout << v[i];
}
```

用有符号的 `z` 后缀 + `ssize()`（C++20 引入的有符号 size 函数）可以自然地写出正确的倒序循环：

```cpp
for (auto i = std::ssize(v) - 1z; i >= 0; --i) { // 有符号运算，i 可以正常变成 -1 从而正确退出循环
    std::cout << v[i];
}
```

这里 `1z` 保证字面量 `1` 的类型和 `std::ssize(v)`（返回有符号类型）在减法运算中类型对齐，避免了混合有符号/无符号运算导致的隐式转换问题。

### 3. 配合 `auto` 做类型推导时更精确

```cpp
auto size = 100uz;      // std::size_t
auto index = -1z;       // 有符号，可以合法表示负数索引偏移之类的中间计算
```

## 和已有后缀的对比

| 后缀     | 类型                    | 是否保证等于 size_t 宽度               |
| -------- | ----------------------- | -------------------------------------- |
| `u`      | `unsigned int`          | 否                                     |
| `ul`     | `unsigned long`         | 否（平台相关）                         |
| `ull`    | `unsigned long long`    | 否（通常够宽，但类型不是 size_t 本身） |
| **`uz`** | **`std::size_t`**       | **是（就是它本身）**                   |
| **`z`**  | **有符号版本的 size_t** | **是（宽度一致）**                     |

## 小结

`z`/`Z` 后缀解决的是一个长期存在但容易被忽视的**类型精确性**问题：C++ 标准库大量使用 `size_t` 作为大小/索引类型（`.size()`、`.length()` 等），但语言层面却一直缺少对应的字面量后缀，导致开发者要么依赖不精确的 `u`/`ul`/`ull`，要么写啰嗦的显式转换。C++23 补齐了这块拼图，同时通过引入**有符号版本**的 `z` 后缀，为"用有符号类型安全地处理容器大小/索引计算、避免无符号下溢"这一常见最佳实践提供了原生语法支持。
