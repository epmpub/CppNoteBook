#### std::set 和 std::vector

| 特性             | std::vector                                             | std::set                                                  |
| ---------------- | ------------------------------------------------------- | --------------------------------------------------------- |
| 容器类型         | 顺序容器（Sequence Container）                          | 关联容器（Associative Container，有序集合）               |
| 元素是否允许重复 | 允许重复                                                | 不允许重复（自动去重）                                    |
| 元素是否自动排序 | 不排序（按插入顺序或你手动管理）                        | 自动按键值排序（默认从小到大，使用 std::less）            |
| 底层实现         | 动态数组（连续内存）                                    | 红黑树（平衡二叉搜索树，非连续内存）                      |
| 随机访问         | 极快（O(1)），支持 v[i]                                 | 不支持（只能通过迭代器顺序访问）                          |
| 查找元素         | std::find 是 O(n) 线性查找                              | set.find() 是 O(log n) 对数查找（快很多）                 |
| 插入元素         | 尾部插入 push_back 很快（摊销 O(1)） 中间插入慢（O(n)） | 任意位置插入都是 O(log n)                                 |
| 删除元素         | 尾部删除快，中间删除慢（O(n)）                          | 删除指定元素 O(log n)                                     |
| 内存布局         | 连续内存（缓存友好，遍历极快）                          | 非连续内存（每个节点分开分配，缓存不友好）                |
| 迭代器失效       | 插入/删除可能导致迭代器/引用失效                        | 插入/删除仅使被删除元素的迭代器失效，其他保持有效         |
| 内存占用         | 较低（只存数据 + 少量额外空间）                         | 较高（每个元素需要额外指针开销，大约是 vector 的 2~3 倍） |
| 典型使用场景     | 需要随机访问、频繁尾部增删、存储大量数据                | 需要自动去重 + 保持有序 + 快速查找                        |

2. 简单记忆口诀

- vector：像动态数组，擅长“我有位置，你给我值”（随机访问强）。
- set：像有序不重复集合，擅长“我有值，你告诉我是否存在”（查找 + 排序强）。
- 代码示例对比

```cpp
#include <vector>
#include <set>
#include <iostream>

int main() {
    std::vector<int> v = {3, 1, 3, 2};
    std::set<int>    s = {3, 1, 3, 2};

    std::cout << "vector: ";
    for (int x : v) std::cout << x << " ";        // 输出: 3 1 3 2   （保留重复，按插入顺序）

    std::cout << "\nset:    ";
    for (int x : s) std::cout << x << " ";        // 输出: 1 2 3      （自动去重 + 排序）
}
```

4. 实际选择建议优先使用 std::vector 的场景：

- 需要随机访问（如 v[100]）
- 数据量很大（百万级以上）
- 主要在尾部增删数据
- 只需要简单列表，不关心去重和顺序

优先使用 std::set 的场景：

- 需要自动去重
- 需要保持有序（方便范围查找、上下界 lower_bound / upper_bound）
- 查找操作非常频繁（查找速度远快于 vector + sort + binary_search）
- 数据量不是特别大（几十万以内）

进阶替代方案（现代 C++ 常用）：

- 只去重、不需要有序 → 用 std::unordered_set（哈希表，平均 O(1) 查找）
- 想兼顾 vector 的连续内存 + set 的去重有序 → 先插入 vector，再排序 + unique，或使用 absl::btree_set（Abseil 库，性能通常更好）

一句话总结：

- vector 是性能优先的动态数组，连续内存，随机访问快。
- set 是功能优先的有序集合，自动去重 + 排序 + 快速查找，但内存和遍历稍慢。
