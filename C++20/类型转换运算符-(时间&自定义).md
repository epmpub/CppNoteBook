### 类型转换运算符

`std::chrono::year_month_day::operator sys_days()` 和 `operator local_days()` 是 C++20 `<chrono>` 日历库提供的**类型转换运算符（conversion operator）**。

它们的作用是：

> 将一个人类可读的日期（年/月/日）转换成一个以天数为单位的时间点（time_point）。

------

## year_month_day 是什么

例如：

```cpp
using namespace std::chrono;

year_month_day ymd{
    year{2026},
    month{6},
    day{4}
};
```

内部保存的是：

```text
2026
6
4
```

这种"日历形式"。

------

## sys_days 是什么

定义大致为：

```cpp
using sys_days =
    sys_time<days>;
```

而：

```cpp
using sys_time =
    time_point<system_clock>;
```

所以：

```cpp
sys_days
```

本质上是：

```cpp
time_point<system_clock, days>
```

表示：

> 从 Unix Epoch（1970-01-01）开始计算的第 N 天。

------

## operator sys_days()

标准定义类似：

```cpp
constexpr operator sys_days() const noexcept;
```

允许：

```cpp
using namespace std::chrono;

year_month_day ymd{2026y, June, 4d};

sys_days sd = ymd;
```

编译器实际上调用：

```cpp
sys_days sd = ymd.operator sys_days();
```

------

### 示例

```cpp
#include <chrono>
#include <iostream>

int main()
{
    using namespace std::chrono;

    year_month_day ymd{2026y, June, 4d};

    sys_days sd = ymd;

    std::cout
        << sd.time_since_epoch().count()
        << '\n';
}
```

输出类似：

```text
20613
```

表示：

```text
1970-01-01 + 20613 天
```

------

## 反向转换

同样支持：

```cpp
sys_days sd = sys_days{2026y/June/4};

year_month_day ymd{sd};
```

即：

```text
日期
↓
sys_days
↓
日期
```

双向转换。

------

## 为什么需要 sys_days

因为日期运算更容易。

例如：

```cpp
auto tomorrow =
    sys_days{2026y/June/4} + days{1};
```

得到：

```text
2026-06-05
```

如果直接操作：

```cpp
year_month_day
```

跨月、跨年会很复杂。

------

## local_days 是什么

定义：

```cpp
using local_days =
    local_time<days>;
```

本质：

```cpp
time_point<local_t, days>
```

注意：

```cpp
local_t
```

不是：

```cpp
system_clock
```

------

区别：

### sys_days

有真实时间轴意义

```text
UTC时间
1970-01-01
1970-01-02
...
```

全世界统一。

------

### local_days

只有本地日历意义

```text
2026-06-04
```

不关联时区。

不知道：

```text
UTC+8
UTC+9
UTC-5
```

是哪一个。

------

## operator local_days()

定义：

```cpp
constexpr operator local_days() const noexcept;
```

例如：

```cpp
using namespace std::chrono;

year_month_day ymd{2026y, June, 4d};

local_days ld = ymd;
```

实际：

```cpp
local_days ld =
    ymd.operator local_days();
```

得到：

```text
2026-06-04 本地日期
```

但没有 UTC 信息。

------

## sys_days vs local_days

假设：

```cpp
2026-06-04
```

------

### sys_days

表示：

```text
UTC基准下的第20613天
```

可以与：

```cpp
system_clock::now()
```

直接比较。

------

### local_days

表示：

```text
本地日历上的2026-06-04
```

不能直接知道对应 UTC 的哪一天。

需要时区：

```cpp
zoned_time
```

才能转换。

------

## 实际示例

```cpp
#include <chrono>
#include <iostream>

int main()
{
    using namespace std::chrono;

    year_month_day ymd{2026y, June, 4d};

    sys_days sd = ymd;
    local_days ld = ymd;

    year_month_day back1{sd};
    year_month_day back2{ld};

    std::cout
        << int(back1.year()) << '-'
        << unsigned(back1.month()) << '-'
        << unsigned(back1.day()) << '\n';

    std::cout
        << int(back2.year()) << '-'
        << unsigned(back2.month()) << '-'
        << unsigned(back2.day()) << '\n';
}
```

输出：

```text
2026-6-4
2026-6-4
```

------

## 为什么设计成 operator

因为日期和天数时间点之间的转换非常常见。

于是标准库允许：

```cpp
sys_days sd = ymd;
```

而不是：

```cpp
sys_days sd = to_sys_days(ymd);
```

同样：

```cpp
local_days ld = ymd;
```

这种写法更自然。

从运算符角度看：

```cpp
std::chrono::year_month_day
```

重载了两个类型转换运算符：

```cpp
constexpr operator sys_days() const noexcept;
constexpr operator local_days() const noexcept;
```

属于前面提到的：

```cpp
operator type()
```

这一类运算符重载，即**用户自定义类型转换运算符**。例如：

```cpp
operator bool();
operator int();
operator std::string();
operator sys_days();
operator local_days();
```

它们都允许对象在需要目标类型时自动完成转换。
