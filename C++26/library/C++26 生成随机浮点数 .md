## std::generate_canonical

生成随机浮点数

```c
#include <iostream>
#include <iomanip>
#include <random>

int main()
{
    // 随机种子
    std::random_device rd;

    // Mersenne Twister
    std::mt19937_64 gen(rd());

    // 生成10个 [0,1) 的 double
    for (int i = 0; i < 10; ++i)
    {
        double value =
            std::generate_canonical<double,
                                    std::numeric_limits<double>::digits>(gen);

        std::cout
            << std::fixed
            << std::setprecision(17)
            << value
            << '\n';
    }
}
```

下面是一个符合 C++26 的 `std::generate_canonical` 使用示例。对于用户代码而言，接口没有变化，变化的是标准库内部实现已经采用新的规范来保证结果严格均匀分布于 `[0,1)`。

```cpp
#include <iostream>
#include <iomanip>
#include <random>

int main()
{
    // 随机种子
    std::random_device rd;

    // Mersenne Twister
    std::mt19937_64 gen(rd());

    // 生成10个 [0,1) 的 double
    for (int i = 0; i < 10; ++i)
    {
        double value =
            std::generate_canonical<double,
                                    std::numeric_limits<double>::digits>(gen);

        std::cout
            << std::fixed
            << std::setprecision(17)
            << value
            << '\n';
    }
}
```

------

如果使用 C++23 之前的写法：

```cpp
double value =
    std::generate_canonical<double, 53>(gen);
```

在 C++26 中仍然完全合法：

```cpp
#include <print>
#include <random>

int main()
{
    std::mt19937_64 gen{std::random_device{}()};

    for (int i = 0; i < 5; ++i)
    {
        auto v = std::generate_canonical<double, 53>(gen);

        std::println("{:.17f}", v);
    }
}
```

输出类似：

```text
0.14423462696677916
0.96291744755895824
0.55002275838913455
0.00201641428306673
0.72992218872566217
```

------

如果想验证其范围，可以进行大量测试：

```cpp
#include <cassert>
#include <random>

int main()
{
    std::mt19937_64 gen{std::random_device{}()};

    for (std::size_t i = 0; i < 100'000'000; ++i)
    {
        double v =
            std::generate_canonical<double, 53>(gen);

        assert(v >= 0.0);
        assert(v < 1.0);
    }
}
```

C++26 新规范保证：

```cpp
v >= 0.0
v < 1.0
```

始终成立。

------

在实际项目中，更常见的写法仍然是：

```cpp
std::uniform_real_distribution<double> dist(0.0, 1.0);

double value = dist(gen);
```

而 `std::generate_canonical` 更适合：

- 实现自定义概率分布
- Monte Carlo 模拟
- 数值计算库
- 随机算法框架内部实现

因为它直接提供标准定义的高质量均匀浮点随机数，是构建其他随机分布的基础组件。
