#### 可变参数友元声明

在 C++26 中，通过核心提案 **[P2893R3](https://www.open-std.org/jtc1/sc22/wg21/docs/papers/2024/p2893r3.html)**（由 Jody Hagins 和 Arthur O'Dwyer 提出），标准引入了**可变参数友元声明（Variadic Friends）**。

这一特性允许在类模板中**直接展开一个参数包（Parameter Pack）来批量声明友元（Friend）**。它极大地简化了泛型编程和特定设计模式（如 Passkey 模式、CRTP 模式）中的权限控制。 [1, 2, 3] 

------

## 1. 历史痛点（C++26 之前）

C++11 允许将模板参数直接声明为友元（例如 `friend T;`）。但是，这种语法**不支持参数包展开**。如果你想让一个可变参数模板中的所有类型都成为友元，在 C++26 之前是没有直接语法的。 [3, 4] 

## 过去的蹩脚规避方案：

开发者为了给一组数量不定的类型赋权，不得不采用固定的最大参数量加上默认模板参数的“笨办法”： [1] 

```cpp
// C++26 之前：必须穷举可能用到的最大类型数量
template <typename T1 = void, typename T2 = void, typename T3 = void>
class ResourceManager {
    // 必须逐个声明友元
    friend T1;
    friend T2;
    friend T3;

private:
    void allocate() {}
};
```

这种做法不仅**限定了友元数量的上限**，而且当参数包为空（`void`）时，编译器还会产生不必要的空友元声明。

------

## 2. C++26 的新语法

从 C++26 开始，你只需在友元声明中使用标准的参数包展开语法（`...`）即可： [1, 2] 

```cpp
template <typename... Ts>
class ResourceManager {
    // C++26 完美解决：将参数包 Ts 中的所有类型一次性批量声明为友元！
    friend Ts...; 

private:
    void allocate() {}
};
```

只要你在实例化 `ResourceManager` 时传入了哪些类型，这些类型就能够自由地访问 `ResourceManager` 的所有 `private` 和 `protected` 成员。 [5] 

------

## 3. 典型应用场景

## 场景 A：增强版的 Passkey 模式 (密匙模式)

Passkey 模式常用于精细化控制成员函数的访问权限（比直接让整个类做友元更安全）。在 C++26 中，结合可变参数友元，你可以轻松创建一个**允许多个特定类访问**的通用密匙： [1, 6] 

```cpp
// 定义一个泛型密匙，只允许 Ts... 里的类构造它
template <typename... Ts>
class Passkey {
    friend Ts...; // 只有 Ts... 中的类能调用构造函数
    Passkey() = default;
};

class Server {
private:
    // 该核心方法需要特殊的 Passkey 权限
    // 只有声明在 Passkey 中的组件才能生成这个密匙并调用它
    void shutdown(Passkey<class AdminConsole, class DatabaseMonitor>) {}
};
```

## 场景 B：复杂的 CRTP (奇特重现模板模式) 与消息分发

在构建诸如消息分发器（Dispatcher）或中介者模式时，基类往往需要知道一组派生类或消息类型的具体私有细节，此时可以通过参数包直接打包授权： [7] 

```cpp
template <typename... Handlers>
class Dispatcher {
    friend Handlers...; // 批量赋予所有处理器访问分发器内部状态的权限
    
private:
    void log_internal_state() {}
};
```

------

## 4. 补充改进：逗号分隔的复合友元声明

作为同一提案（P2893R3）的副产物，C++26 还顺带消除了另一个冗余语法：**允许使用逗号在一行代码中声明多个普通的友元类型**。 [4] 

- **C++26 之前**：

  ```cpp
  class Window {
      friend class Button;
      friend class Label;
      friend class Sidebar;
  };
  ```

- **C++26 之后**：

  ```cpp
  class Window {
      // 更加紧凑和现代的现代写法
      friend class Button, Label, Sidebar; 
  };
  ```

------

## 总结

C++26 的 **Variadic Friends** 填补了参数包在“类访问控制（Access Control）”领域的最后一块空白。至此，C++ 的参数包已经可以在继承列表、构造函数初始化列表、结构化绑定以及友元声明中进行全方位展开。 [2, 3]
