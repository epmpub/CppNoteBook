对，Rust 的 `?` 操作符，本质上就是：

```text
自动展开错误传播模板代码
```

它解决的是：

```text
“层层 if (!r) return error;” 的样板代码爆炸
```

这是 Rust 错误处理能真正流行起来的核心原因之一。

------

# 1. 没有 `?` 的世界

先看 C++ `expected` 风格。

------

## C++ today

```cpp
auto r1 = parse();

if (!r1)
    return std::unexpected(r1.error());

auto r2 = validate(*r1);

if (!r2)
    return std::unexpected(r2.error());

auto r3 = save(*r2);

if (!r3)
    return std::unexpected(r3.error());

return *r3;
```

问题：

- 重复
- 噪音巨大
- 淹没业务逻辑
- nesting pain

------

# 2. Rust `?`

Rust：

```rust
fn process() -> Result<Data, Error> {
    let x = parse()?;
    let y = validate(x)?;
    let z = save(y)?;
    Ok(z)
}
```

这就是：

```text
错误自动向上传播
```

------

# 3. `?` 实际展开

Rust：

```rust
let x = parse()?;
```

本质接近：

```rust
let x = match parse() {
    Ok(v) => v,
    Err(e) => return Err(e),
};
```

所以：

```text
? = 自动生成 match + return Err
```

------

# 4. 这解决了什么

这是系统语言几十年的大问题：

------

## C 风格

```c
if (err) return err;
```

到处都是。

Linux kernel 全是：

```c
if (ret < 0)
    return ret;
```

------

## C++ exception

试图解决：

```text
错误传播样板代码
```

于是：

```cpp
foo();
bar();
baz();
```

自动 unwinding。

但：

- hidden control flow
- runtime cost
- ABI complexity
- cache issue
- not deterministic

很多系统领域不喜欢。

------

Rust 做了个非常聪明的折中：

```text
错误是值（Result）
+
传播自动化（?）
```

这就是关键。

------

# 5. 为什么 `expected` 没 Rust 那么爽

因为：

```text
C++ 缺少语言级 error propagation
```

目前只能：

------

## 方案1：手写

```cpp
if (!r)
    return unexpected(r.error());
```

------

## 方案2：monadic chain

```cpp
parse()
.and_then(validate)
.and_then(save);
```

虽然优雅：

但：

- 不适合复杂控制流
- debug 不自然
- 局部变量不方便

------

# 6. 所以 C++ 社区疯狂想要 `?`

这些年很多 proposal：

| Proposal         | 目标                 |
| ---------------- | -------------------- |
| TRY macro        | 模拟 `?`             |
| operator ?       | 真正语法糖           |
| pattern matching | 简化 Result handling |
| monadic expected | 减少 boilerplate     |

------

# 7. 现在很多 C++ 项目已经手搓 `TRY`

例如：

```cpp
#define TRY(x) ({ \
    auto _r = (x); \
    if (!_r) return std::unexpected(_r.error()); \
    *_r; \
})
```

然后：

```cpp
auto x = TRY(parse());
auto y = TRY(validate(x));
```

这其实就是：

```text
山寨 Rust ?
```

Google/Facebook/游戏引擎里很多类似东西。

------

# 8. Rust 真正厉害的地方

不是 Result。

很多语言都有：

- Haskell Either
- Swift Result
- Kotlin Result
- expected
- outcome

真正厉害的是：

```text
Result + ? + pattern matching
```

三件套组合。

于是：

```text
错误处理：
- 显式
- 零成本
- 易传播
- 不啰嗦
```

这是 Rust 非常成功的语言设计。

------

# 9. 为什么 C++ 很难补这个

因为 C++ 有：

- exception
- RAII
- overload rules
- templates
- backward compatibility
- old STL APIs

所以：

```text
没法像 Rust 一样“从零设计”
```

C++ 只能：

```text
一点点 Rust 化
```

于是你会看到：

- `std::expected`
- monadic API
- pattern matching proposal
- sender/receiver
- coroutine

越来越像 Rust/FP。

------

# 10. 现代系统语言趋势

过去：

```text
错误 = side effect
```

现在：

```text
错误 = 类型系统的一部分
```

即：

```rust
Result<T,E>
```

或：

```cpp
expected<T,E>
```

这是最近十多年系统编程最大的范式变化之一。
