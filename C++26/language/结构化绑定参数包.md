

## 结构化绑定参数包

**C++26 Structured Binding Pack（结构化绑定参数包）**，属于 Structured Binding 的增强版本。

```cpp
#include <print>

template<typename T>
void inspect(const T& obj)
{
    // C++26 Structured Binding Pack
    const auto& [...values] = obj;

    std::println("Object has {} elements", sizeof...(values));

    int count = 0;

    (
        std::println("Elem {} : {}", count++, values),
        ...
    );
}

int main()
{
    struct MyData
    {
        int x;
        int y;
        float z;
    };

    MyData value{1, 2, 4.3f};

    inspect(value);

    return 0;
}
```

核心代码：

```cpp
const auto& [...values] = obj;
```

这是 C++26 新增语法。

在 C++17 中，你只能这样写：

```cpp
const auto& [x, y, z] = obj;
```

必须明确写出绑定变量的个数。

而 C++26 引入了 Binding Pack：

```cpp
const auto& [...values] = obj;
```

表示：

> 将对象中的所有成员绑定到一个参数包 `values...` 中。

对于：

```cpp
struct MyData {
    int x;
    int y;
    float z;
};

MyData value{1,2,4.3f};
```

实际上等价于概念上的：

```cpp
auto& [v0, v1, v2] = value;
```

然后：

```cpp
values...
```

代表：

```cpp
v0, v1, v2
```

------

后面的代码：

```cpp
sizeof...(values)
```

统计绑定包中的元素数量：

```cpp
std::print("Object has: {} elements\n",
           sizeof...(values));
```

输出：

```text
Object has: 3 elements
```

------

这里：

```cpp
(std::print("Elem: {} : {}", count++, values), ...);
```

是一个 Fold Expression（折叠表达式）。

展开后相当于：

```cpp
std::print("Elem: {} : {}", count++, v0);
std::print("Elem: {} : {}", count++, v1);
std::print("Elem: {} : {}", count++, v2);
```

输出类似：

```text
Elem: 0 : 1
Elem: 1 : 2
Elem: 2 : 4.3
```

------

这段代码实际上同时演示了两个 C++26 特性：

### 1. Structured Binding Pack

```cpp
const auto& [...values] = obj;
```

允许结构化绑定得到一个参数包。

### 2. Pack Indexing / Pack Expansion

这里使用了：

```cpp
values
```

在 Fold Expression 中展开：

```cpp
(..., values)
```

实现对所有成员的遍历。

------

如果是普通的 C++17 Structured Binding，只能写：

```cpp
auto [x, y, z] = value;
```

而不能写：

```cpp
auto [...values] = value;  // C++26
```

因此回答你的问题：

**是的，这段代码属于 Structured Binding（结构化绑定）的范畴，但更准确地说，它演示的是 C++26 的 Structured Binding Pack，而不是 C++17 的普通结构化绑定。**
