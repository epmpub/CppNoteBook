## Structured Bindings

在 C++26 中，通过核心提案 **[P0609R3](https://www.open-std.org/jtc1/sc22/wg21/docs/papers/2023/p2308r1.html)**，标准正式引入了**为结构化绑定（Structured Bindings）中的“单个独立变量”指定属性（Attributes）**的功能。 [1, 2] 

这是对 C++17 结构化绑定语法的重大补全，极大地提升了编译器警告控制的粒度。 [1, 2] 

------

## 1. 历史痛点（C++26 之前）

在 C++20/23 中，你只能给结构化绑定的**整个语句**（即编译器幕后生成的匿名隐藏对象）添加属性，而**无法**精确控制解构出来的每一个变量： [1] 

```cpp
// C++26 之前：只能标记整个语句
[[maybe_unused]] auto [x, y] = get_pair(); 
```

这带来了一个巨大的痛点：如果你只需要用到 `y`，而不需要用到 `x`（例如 `x` 是一个状态码或日志 ID），在开启 `-Wunused-variable` 编译选项时，编译器会为 `x` 抛出警告。 [2, 3] 

- 如果你试图在 `x` 前面加属性：`auto [[[maybe_unused]] x, y] = ...` -> **编译报错（语法非法）**。
- 如果你标记整个语句 `[[maybe_unused]]`，虽然隐瞒了 `x` 没被使用的警告，但**同时也让 `y` 失去了未使用的警告保护**。如果后续代码不小心删掉了对 `y` 的使用，编译器将无法提醒你。 [1, 2] 

------

## 2. C++26 的新语法

从 C++26 开始，你可以在结构化绑定的中括号内部，**直接为任意一个或多个标识符单独附加属性**。 [1, 2] 

## 核心应用场景：精确控制 `[[maybe_unused]]`

```cpp
#include <map>
#include <string>
#include <iostream>

int main() {
    std::map<int, std::string> my_map{{1, "one"}, {2, "two"}};

    // 完美解决：仅将 key 标记为可能未被使用，保持对 value 的未使用检查
    for (const auto& [[[maybe_unused]] key, value] : my_map) {
        std::cout << value << "\n"; 
    }
}
```

属性的放置位置可以位于**变量名之前**，也可以位于**变量名之后**： [1] 

```cpp
auto [ [[maybe_unused]] x, y ] = get_pair(); // 放在前面（推荐，更符合直觉）
auto [ x, y [[maybe_unused]] ] = get_pair(); // 放在后面亦可
```

------

## 3. 为什么这个改动等了这么久？

允许在 `[...]` 中使用属性在语法解析上非常复杂。因为 C++ 属性使用双中括号 `[[attr]]`，而结构化绑定使用单中括号 `[...]`。当两者嵌套时，编译器必须精准区分诸如 `[ [[attr]] x ]` 这种多重括号的边界，早期的编译器前端解析器对这种语法的支持存在技术困难。 [3] 

随着 C++26 编译器技术的成熟，该语法已经得到了各大主流编译器的支持（如 **GCC 15+** 和 **Clang 19+** 已实现该特性）。 [4, 5] 

------

## 4. 配合 C++26 另一个新特性的妙用

C++26 同时引入了**无名占位符 `_`（Unnamed Placeholder - P2169R4）**。这两个特性在处理不需要的解构变量时各司其职： [6] 

- **如果你完全不需要这个变量**：直接使用下划线 `_` 占位，它会自动被编译器忽略且不占用命名空间。

  ```cpp
  auto [_, value] = get_pair(); // C++26 无名占位符
  ```

- **如果变量在某些宏或特定编译条件下才被使用**（例如仅在 `DEBUG` 宏下被使用）：此时**无法**使用 `_`，必须使用属性：

  ```cpp
  auto [ [[maybe_unused]] id, value ] = get_pair();
  assert(id > 0); // 在 Release 模式下 assert 被消除，id 会变成未使用，但有属性保护不会报错
  ```
