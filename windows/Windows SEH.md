# 结构化异常处理（SEH）

**MSVC 的 `__try` 是****结构化异常处理（SEH）**，专门抓**系统级崩溃**（除零、空指针、越界）；

**C++ 标准 `try/catch` 是****C++ 异常处理**，专门抓 **throw 抛出的异常**。

**除了捕获异常类型不同，它们在功能、机制、用途上完全不是一回事。**

------

# 一、最核心区别（必须记住）



|       特性       |          **__try / __except (SEH)**          |           **try /catch (标准 C++)**            |
| :--------------: | :------------------------------------------: | :--------------------------------------------: |
|   **捕获对象**   | **系统异常**（崩溃：空指针、除零、非法访问） | **C++ 异常**（`throw int`、`throw exception`） |
|   **异常来源**   |           Windows 操作系统内核发出           |               代码里手动 `throw`               |
| **能否互相捕获** |            **不能捕获 C++ throw**            |              **不能捕获系统崩溃**              |
|   **适用平台**   |               **Windows 专属**               |                   全平台通用                   |
|   **功能重点**   |          进程崩溃保护、恢复、错误码          |         面向对象异常、错误类型、栈展开         |
|    **栈展开**    |         **默认不展开**（不析构对象）         |           **自动栈展开**（自动析构）           |

------

# 二、__try（SEH）**独有的强大功能**（标准 try 没有）

这是你问的重点：**除了 SEH，__try 还有哪些专属功能？**

## 1. 可以捕获**硬件异常 / 系统崩溃**

标准 try **完全做不到**：

```
// 下面这些崩溃，标准 try 抓不到
// 只有 __try 能抓

int* p = nullptr;
*p = 10;  // 空指针访问 → 崩溃

int a = 10 / 0; // 除零 → 崩溃
```

## 2. 提供 **异常代码（Exception Code）**

能精准知道崩溃原因：

- `EXCEPTION_ACCESS_VIOLATION` 空指针
- `EXCEPTION_INT_DIVIDE_BY_ZERO` 除零
- `EXCEPTION_STACK_OVERFLOW` 栈溢出

标准 try 没有这个能力。

## 3. 支持 **三种异常处理策略**



```
__except(EXCEPTION_EXECUTE_HANDLER)    // 我来处理
__except(EXCEPTION_CONTINUE_SEARCH)    // 交给上层
__except(EXCEPTION_CONTINUE_EXECUTION) // 忽略错误继续执行
```

**标准 C++ 没有这种控制能力。**

## 4. **__finally** 无论是否异常，一定执行（SEH 独有）

cpp



运行









```
__try {
}
__finally {
    // 不管是正常退出、异常退出、return、goto
    // 这里 100% 会执行
}
```

作用：**保证资源释放**，比 `delete` 更可靠。

⚠️ 注意：C++ 没有 `finally`，只能靠析构函数。

## 5. 能捕获 **Windows 系统服务 / 驱动级异常**

SEH 是 Windows 系统原生机制，深入内核。

------

# 三、标准 try/catch **独有的功能**（SEH 没有）

1. 捕获任意 C++ 类型（`int`、`string`、`class`）
2. 多类型 catch 匹配
3. **自动栈展开（自动析构局部对象）**
4. 全平台通用（Linux/macOS 也能用）

------

# 四、最关键的大坑（99% 的人踩过）

## 1. **SEH 不会触发 C++ 析构函数！**

cpp



运行









```
void test() {
    MyClass obj;  // 有析构函数

    __try {
        crash();
    }
    __except(1) {
    }

    // obj **不会被析构！！！** 内存泄漏
}
```

⚠️ **SEH 默认不做栈展开**

⚠️ **C++ try 会自动析构**

------

# 五、能不能一起用？

可以，但**必须小心**：

### 微软提供一个函数：

cpp



运行









```
_set_se_translator(...);
```

作用：

**把系统异常（SEH）转换成 C++ 异常**

让你能用 `try/catch` 捕获空指针、除零。

------

# 六、最终总结（最精简）

### **__try / __except（SEH）**

- **Windows 专属**
- 抓**系统崩溃**（空指针、除零、非法访问）
- 有 `__finally`
- 有异常错误码
- **不自动析构对象**
- 用于**保护进程不崩溃**

### **try /catch（标准 C++）**

- 全平台通用
- 抓 **throw**
- **自动析构对象**
- 用于**业务错误处理**

------

### 一句话终极答案

__try 除了处理 SEH 系统异常，还提供 finally 保证执行、异常错误码、崩溃恢复；

而标准 try 只能处理 C++ throw，不能处理系统崩溃。
