#### `SetProcessWorkingSetSizeEx` + `VirtualLock` 来尝试将大块内存锁定在物理 RAM 中

```cpp
#include <windows.h>
#include <iostream>
#include <iomanip>
#include <vector>

int main()
{
    // ==================== 1. 提升进程权限和优先级 ====================
    SetPriorityClass(GetCurrentProcess(), REALTIME_PRIORITY_CLASS);

    // ==================== 2. 设置最小/最大工作集（关键步骤） ====================
    // 建议：最小工作集设得比你要锁定的内存稍大一些
    //       最大工作集可以设得很大或也设为相同值
    const SIZE_T targetMinWS = 2ULL * 1024 * 1024 * 1024;   // 2 GB 示例
    const SIZE_T targetMaxWS = 4ULL * 1024 * 1024 * 1024;   // 4 GB 示例

    BOOL ret = SetProcessWorkingSetSizeEx(
        GetCurrentProcess(),
        targetMinWS,                    // dwMinimumWorkingSetSize
        targetMaxWS,                    // dwMaximumWorkingSetSize
        QUOTA_LIMITS_HARDWS_MIN_ENABLE | QUOTA_LIMITS_HARDWS_MAX_ENABLE
        // 或者只用 QUOTA_LIMITS_HARDWS_MIN_ENABLE（更常用）
    );

    if (!ret) {
        std::cerr << "SetProcessWorkingSetSizeEx 失败，错误码: " 
                  << GetLastError() << std::endl;
        // 常见错误：5 (ACCESS_DENIED) → 需要以管理员身份运行
    } else {
        std::cout << "SetProcessWorkingSetSizeEx 成功：Min = " 
                  << (targetMinWS / (1024*1024)) << " MB, Max = " 
                  << (targetMaxWS / (1024*1024)) << " MB" << std::endl;
    }

    // ==================== 3. 分配内存并锁定 ====================
    const SIZE_T chunkSize = 512 * 1024 * 1024;   // 每次尝试锁定 512 MB（可调整）
    std::vector<void*> lockedRegions;
    SIZE_T totalLocked = 0;

    std::cout << "开始分配并尝试锁定内存...\n";

    while (true) {
        // 1. 分配并提交内存
        void* pMem = VirtualAlloc(
            NULL,
            chunkSize,
            MEM_COMMIT | MEM_RESERVE,
            PAGE_READWRITE
        );

        if (!pMem) {
            std::cerr << "VirtualAlloc 失败（commit limit 已满），错误码: " 
                      << GetLastError() << std::endl;
            break;
        }

        // 2. 写入数据（让页面变成 dirty，不可压缩）
        memset(pMem, 0xAA, chunkSize);   // 或逐字节写入更复杂的模式

        // 3. 尝试锁定到物理内存
        if (VirtualLock(pMem, chunkSize)) {
            lockedRegions.push_back(pMem);
            totalLocked += chunkSize;
            std::cout << "成功锁定 + " << (chunkSize / (1024*1024)) 
                      << " MB，总计锁定: " << (totalLocked / (1024*1024)) 
                      << " MB" << std::endl;
        } else {
            DWORD err = GetLastError();
            if (err == ERROR_WORKING_SET_QUOTA) {  // 1453
                std::cerr << "工作集配额不足（ERROR_WORKING_SET_QUOTA），无法继续锁定。\n";
                std::cerr << "当前已锁定: " << (totalLocked / (1024*1024)) << " MB\n";
            } else {
                std::cerr << "VirtualLock 失败，错误码: " << err << std::endl;
            }
            VirtualFree(pMem, 0, MEM_RELEASE);  // 释放未锁定的内存
            break;
        }
    }

    std::cout << "\n=== 锁定完成 ===\n";
    std::cout << "总共成功锁定物理内存: " << (totalLocked / (1024*1024)) << " MB\n";
    std::cout << "按任意键退出（程序退出时会自动解锁）...\n";
    std::cin.get();

    // ==================== 4. 清理（可选） ====================
    for (auto p : lockedRegions) {
        VirtualUnlock(p, chunkSize);   // 虽然退出时会自动解锁
        VirtualFree(p, 0, MEM_RELEASE);
    }

    return 0;
}
```

### 使用注意事项（非常重要）

1. **必须以管理员身份运行**（右键 → 以管理员身份运行），否则 `SetProcessWorkingSetSizeEx` 常返回 ACCESS_DENIED。
2. **锁定上限**：
   - 能成功锁定的最大内存 ≈ **最小工作集大小** 减去少量系统开销。
   - 即使你把最小工作集设成 8GB，在桌面版 Windows 上通常也只能实际锁定几 GB（具体取决于物理内存、Windows 版本、pagefile 等）。
   - 很快就会遇到 **ERROR_WORKING_SET_QUOTA (1453)**。
3. **Flags 参数推荐**：
   - `QUOTA_LIMITS_HARDWS_MIN_ENABLE`：强制最小工作集生效（最常用）。
   - `QUOTA_LIMITS_HARDWS_MAX_ENABLE`：强制最大工作集生效（会更严格限制，但可能导致更多 page fault）。
4. **对系统的影响**：
   - 大量使用 `VirtualLock` 会严重占用物理内存，导致其他程序频繁换页，系统整体变慢甚至卡顿。
   - Microsoft 官方明确警告：这可能会降低系统性能。
5. **监控工具**：
   - **Process Explorer**（Sysinternals） → 查看 Working Set 和 Private Bytes。
   - **VMMap**（Sysinternals）可看到 “Locked WS” 列（普通工具看不到锁定的用户态页面）。

### 实际效果预期
- 在 16GB~32GB 内存的机器上，你可能成功锁定 1~4GB（视具体配置而定）。
- 继续尝试会因工作集配额耗尽而停止。
- 这比单纯 `VirtualAlloc + memset` 更能让内存**常驻物理 RAM**，但仍然无法“吃满”整个系统内存。
