在 C++ 函数编程中，柯里化（Currying）是一种将接受多个参数的函数转换为一系列接受单个参数的函数的技术。它主要解决了以下几个问题：

1. **参数复用与固定**柯里化允许固定函数的部分参数，生成一个新的函数。这在需要多次调用同一函数但部分参数不变的场景中非常有用，避免了重复传递相同参数。例如：

   ```cpp
   // 原函数：计算 a*b + c
   int func(int a, int b, int c) { return a*b + c; }
   
   // 柯里化后固定 a=2，生成新函数
   auto curried = [](int a) {
       return [a](int b) {
           return [a, b](int c) {
               return a*b + c;
           };
       };
   };
   
   auto f = curried(2);  // 固定 a=2
   f(3)(4);  // 等价于 func(2,3,4)
   ```

   

2. **延迟执行与分步计算**柯里化将多参数函数的执行拆分为多个步骤，每个步骤接受一个参数并返回下一个函数。这使得可以分阶段处理参数，适合需要动态获取参数的场景（如用户输入逐步传递）。

3. **函数组合与高阶函数适配**柯里化后的函数更容易与其他函数组合，尤其在函数式编程中，许多高阶函数（如 `map`、`filter`）期望接受单参数函数。柯里化可以将多参数函数转换为符合要求的形式。例如：

   ```cpp
   // 柯里化的加法函数
   auto add = [](int a) { return [a](int b) { return a + b; }; };
   
   std::vector<int> nums = {1,2,3};
   // 使用柯里化函数适配 std::transform（需要单参数函数）
   std::transform(nums.begin(), nums.end(), nums.begin(), add(5));
   // 结果：{6,7,8}
   ```

   

4. **类型推导与接口简化**在模板编程中，柯里化可以帮助编译器更轻松地推导类型，同时简化函数接口，使代码更具可读性和扩展性。

柯里化本质上是将函数的 "多参数依赖" 转化为 "单参数链式依赖"，从而增强函数的灵活性和复用性，尤其在函数式编程范式中能显著提升代码的表达力。
